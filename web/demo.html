DÉMO · PITCH PRODUIT

De l'intention à l'artefact.
En une frappe.

Suis le parcours d'une demande, de ta phrase jusqu'à la preuve scellée.

01 · LE PROBLÈME

Les agents IA sont devenus une soupe.

30 outils. 0 traçabilité. 0 reproductibilité. Chaque agent répond dans sa propre bulle, sans mémoire partagée, sans contrat, sans preuve.

Le résultat : des pipelines qu'on ne peut pas auditer, des artefacts qu'on ne peut pas rejouer, des équipes qui recommencent à zéro à chaque run.

02 · LA CONVICTION

"Un noyau. Une trace. Une forge."

Grimoire Kit part d'un principe simple : le runtime doit être le seul point de vérité. Tous les agents opèrent sous son contrôle. Tous les artefacts sortent par son registre.

Pas d'appels API dispersés. Pas de mémoire locale non partagée. Un noyau unique, traçable, composable.

03 · LE SOG

Un seul point d'entrée. Des dizaines de sub-agents invisibles.

Le Smart Orchestrator Gateway reçoit la demande, analyse le contrat, dispatche vers les agents spécialisés, agrège les résultats.

L'opérateur parle à une interface cohérente. La complexité d'orchestration est entièrement encapsulée dans le runtime.

04 · LA FRAPPE

Chaque artefact est forgé, pas généré.

Une frappe d'artefact n'est pas un texte retourné par un LLM. C'est un objet versioned, adressable par hash, signé par le contrat qui l'a produit.

L'artefact porte en lui la preuve de sa fabrication : qui l'a produit, quand, avec quels outils, sous quel contrat.

05 · LA TRACE

Chaque milliseconde est enregistrée.

Le trace ledger capture chaque appel LLM, chaque outil invoqué, chaque transfert entre agents, chaque décision du SOG.

La trace est immutable, exportable en JSONL, rejoutable à l'identique. C'est la preuve que les auditeurs attendent.

06 · LES SURFACES

Une surface par usage. Un seul runtime.

CLI et SDK Python pour piloter. Serveur MCP pour brancher Copilot ou Claude Desktop. Observatory pour l'audit, Kanban gouverné pour le flux, Game UI pour la lisibilité.

Toutes lisent le même runtime — la trace, le task-board, le registre d'artefacts. Le Cockpit temps réel, lui, reste une vision assumée.

07 · L'OBSERVABILITÉ

Observer, pas deviner.

Latence, coût, tokens, hallucinations, profondeur d'orchestration. Métriques temps réel, rétention longue, alertes configurables.

Une observability conçue pour un runtime agentique, pas pour des VMs. Chaque métrique a un sens dans le contexte du SOG.

08 · LE GAME UI

Le runtime rendu visible.

Chaque agent est un personnage dans un bureau isométrique. Chaque tâche devient une animation. Ce n'est pas du cosmétique : c'est une vraie surface d'observation.

Les anomalies se voient avant que les métriques les capturent. L'onboarding s'accélère. L'abstraction disparaît.

09 · POUR QUI

Trois profils. Un noyau.

Solo builder qui veut un runtime prêt en 48h. Équipe produit qui exige la traçabilité. Plateforme interne qui doit exposer des agents à des équipes métier.

Grimoire Kit s'adapte au contexte sans changer de paradigme. Le contrat est toujours le même : trace, artefact, preuve.

10 · DÉMARRER

Trois étapes. Premier artefact en 15 minutes.

Clone, configure, forge. La CLI prend en charge l'initialisation du runtime, la définition du premier contrat et le spawn des agents.
