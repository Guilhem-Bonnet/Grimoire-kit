Extensions

Chaque extension packe un framework agentique public en artefacts gouvernés — agents, skills, hooks, workflows — installés dans votre projet et ancrés sur le catalogue de patterns. Pas de boîte noire : un manifeste validé, des permissions déclarées, des hooks en mode shadow.

Manifeste validé

Chaque extension déclare un extension.json validé par schéma avant toute installation. Chemins relatifs uniquement, pas de remontée hors du bundle.

Ancrage patterns

Le mapping vers le catalogue de patterns est obligatoire. Une extension sans patterns.implements est refusée — c'est ce qui distingue ce marketplace d'un annuaire.

Hooks en shadow

Tout hook fourni s'enregistre en mode shadow, sans exception. La promotion vers un mode bloquant reste une décision du projet hôte.

Permissions déclarées

Filesystem, réseau, événements interceptés, accès mémoire : tout est déclaré dans le manifeste et vérifiable avant installation.
