Le runtime, rendu visible.

Chaque agent devient un personnage. Chaque relation, un flux. Chaque trace, une archive. La scène est peuplée par les vraies données de observatory.py.

01 · LÉGENDE

Agents

Chaque personnage est un agent du registre (persona + capacités). Son badge indique son nombre de traces ; son halo, son activité récente.

Flux

Les lignes entre agents sont les relations du graphe d'agents. L'épaisseur reflète la force du lien ; survolez un agent pour isoler ses flux.

handoff spec mémoire escalade

Traces

Chaque agent porte ses traces réelles (ACTION, DECISION, HANDOFF…). Ouvre un personnage pour voir ses dernières traces — les mêmes que dans l'Observatory.

02 · POURQUOI UN GAME UI
01

Lisibilité émotionnelle

Un tableau de bord texte demande de l'interprétation. Une scène isométrique se lit d'un coup d'œil. L'état du runtime devient intuitif, même pour un non-technicien.

02

Détection d'anomalies

Un agent immobile trop longtemps, un bureau vide alors que le pipeline tourne — ces signaux visuels alertent avant même que l'Observatory les détecte en métrique.

03

Onboarding accéléré

Comprendre qu'un agent « passe le résultat » à un autre, c'est abstrait. Le voir relié par un flux, c'est immédiat. La courbe d'apprentissage s'effondre.

Observer le runtime en temps réel.
