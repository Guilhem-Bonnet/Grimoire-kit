FORGE-05 · GAME UI · BETA

Le runtime, rendu visible.

Chaque agent devient un personnage. Chaque tâche, une scène. Chaque trace, une archive.

TIMELINE — 60 DERNIÈRES SECONDES T+0s
01 · LÉGENDE

Agents

Chaque personnage représente un agent actif. Sa couleur indique son rôle. Son animation indique son état : idle, working, waiting, done.

Artefacts

Les artefacts apparaissent sur le bureau de l'agent qui les a forgés. Un flash orange indique l'émission. L'artefact reste jusqu'au prochain cycle.

Traces

Les lignes entre agents représentent les appels actifs. Chaque flux correspond à un event dans le trace ledger, visible dans l'Observatory.

02 · POURQUOI UN GAME UI
01

Lisibilité émotionnelle

Un tableau de bord texte demande de l'interprétation. Une scène isométrique se lit d'un coup d'œil. L'état du runtime devient intuitif, même pour un non-technicien.

02

Détection d'anomalies

Un agent immobile trop longtemps, un bureau vide alors que le pipeline tourne — ces signaux visuels alertent avant même que l'Observatory les détecte en métrique.

03

Onboarding accéléré

Comprendre qu'un agent "passe le résultat" à un autre agent, c'est abstrait. Le voir marcher entre deux bureaux, c'est immédiat. La courbe d'apprentissage s'effondre.
