01 · PHILOSOPHIE
01 · SDK, PAS UN SaaS

S'installe avec pip. Tourne dans ton projet.

Aucune dépendance cloud, aucun vendor lock-in. Le runtime s'exécute localement — dans ton IDE, dans ton CI, dans ton infra. Tu gardes le contrôle complet de chaque artefact produit.

02 · UN SEUL POINT D'ENTRÉE

Le SOG dispatche. Tu ne parles qu'à un agent.

Le Smart Orchestrator Gateway analyse l'intention, dispatche vers des sub-agents invisibles, agrège les résultats. L'opérateur contrôle un système cohérent, pas une collection d'appels API.

03 · COMPOSABLE PAR NATURE

Agents, skills, workflows : tout est modulaire, forgeable.

Les modules BMM, CIS, TEA, BMB se combinent librement. Chaque agent est un fichier Markdown versionné. L'UDF crée dynamiquement ce qui manque. Tout est remplaçable sans toucher au runtime.

02 · MODULES
